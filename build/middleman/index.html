Middleman

Static Website Engine

Wir nutzen zur Erstellung statischer HTML Sites den Website Generator Middleman. Lesen Sie hier, welche Vorteile ein Static Website Generator bietet und warum die Kosten für Erstellung und Betrieb von Websites enorm reduziert werden können.

In letzter Zeit liest man immer wieder von einer Renaissance statischer HTML Sites. Schaut man sich Statistiken an, stimmt das nicht so richtig. Der Anteil statischer HTML Sites ist über all die Jahre nahezu gleich geblieben (ca. 60%).

Allerdings gibt es eine Renaissance bei Entwicklern. Immer mehr setzen zum Schreiben ihrer Blogs sog. statische Website Generatoren ein. Der bekannteste ist Jekyll. Ich setze hier Middleman ein. Jekyll ist mehr für Blogs geeignet und eher nicht für normale Sites, wie z.B. eine Firmensite.

Die Tatsache, dass solche Systeme von Programmierern genutzt werden, ist auch deren Nachteil - man muss zumindest über ein wenig technisches Verständnis verfügen und HTML und CSS beherrschen.

Falls Sie jetzt denken: dann kommt das für mich nicht in Frage, lesen Sie bitte weiter. Vielleicht kann ich Sie ja vom Gegenteil überzeugen.

Schauen wir uns einmal die Vorteile an:

  • Höchstmögliche Geschwindigkeit - bei immer mehr Aufrufen von mobilen Geräten und der Tatsache, dass Google Geschwindigkeit, als Kriterium zur Bewertung von Websites nutzt, ist Geschwindigkeit ein wesentlicher Faktor.
  • geringe Server Last, keine Datenbank Abfragen, keine dynamische Generierung des ausgelieferten HTML.
  • einfacher zu erweitern und zu bedienen als traditionelle CMS
  • Höchste Sicherheit, da z.B. keine anfälligen, programmierte Formulare enthalten sind
  • keine serverseitige Anforderungen wie PHP, Java, Ruby, etc.

Statische Website Generatoren funktionieren so:

  • man schreibt seine Inhalte und speichert diese in einem speziellen Verzeichnis auf dem eigenen Rechner.
  • ein einfacher Befehl muss im Terminal eingegeben werden und der Generator generiert das HTML in einem anderen speziellen Verzeichnis.
  • dieses könnte man jetzt per FTP auf den Online Server übertragen, aber es geht viel besser.

Wie zu Beginn dieses Artikels geschrieben, werden und wurden schon immer die meisten Websites als statisches HTML ausgeliefert. Aber schon immer war die Erstellung und Pflege höchst unkomfortabel. Eine Seitennavigation bspw. muss auf jeder Seite, meisten per Copy & Paste, neu eingefügt werden. Möchte man einen Navigationspunkt ändern oder hinzufügen, muss man das auf jeder Seite durchführen. Das mag auf kleinen Sites mit 10 bis 20 Seiten noch möglich sein, aber irgendwann ist das nicht mehr machbar.

Zwar gab es schon immer HTML Editoren, die hier mehr oder weniger komfortable Möglichkeiten boten das Problem zu lösen, aber wirklich befriedigend war das nie.

Und tatsächlich wurden serverseitige Programmiersprachen u.A. eingeführt, um solche Problem zu lösen. Viele Sites werden, nur wegen dieser Probleme, heute noch mit PHP entwickelt und betrieben.

Hier kommt die am Anfang genannte Renaissance statischer Websites ins Spiel. Viele Programmierer haben eben erkannt, dass in vielen Fällen ein statischer Website Generator, der sog. serverseitigen Programmierung vorzuziehen ist.

Würden Sie mich z.B. mit der Erstellung einer statischen Site, Inhaltserstellung und weiteren Pflege beauftragen, würde ich den von mir bevorzugten Generator Middleman einsetzen.

  • Die Erstellung geht schneller, einfacher und kostengünstiger.
  • Die Veröffentlichung neuer oder geänderter Inhalte geschieht beinahe vollautomatisch.
  • Ein Versionskontrollsystem wie git sichert alle Versionen der Inhalte.
  • Man kann jederzeit zu einer vorherigen Version zurückkehren.
  • Gleichzeitig kann mit git die Veröffentlichung der Inhalte im Live System angestossen werden.
  • Das Hosting kann kostenlos sein.

Und last but not least können Sie die Inhalte Ihrer statischen Website mit einem CMS pflegen. Das mache ich hier auf mdwp* mit allen neuen Inhalten und zwar mit Contentful als CMS. Und obwohl die Site statisch ist, gibt es Kommentarmöglichkeiten für Besucher, ein Kontaktformular (auch eine Vielzahl von Formularen wäre möglich) und ein Teaser von jedem neuen Inhalt wird automatisch nach Facebook, Twitter und Linkedin gesendet.

Alles ohne zusätzlichen Aufwand.
