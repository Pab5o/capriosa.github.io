Backdrop

Drupal 7 Fork

Einige Drupal Entwickler sind nach langen Überlegungen zu dem Ergebnis gekommen, dass der Weg, der mit Drupal 8 eingeschlagen wurde, für kleine und mittlere Sites, nicht der richtige ist. Deshalb wurde ein Drupal 7 Fork, als Alternative zu D7 und D8, entwickelt.

Was ist das Backdrop CMS?

  • Ein Content Management System
  • Ein Programmier Framework
  • Ein Drupal 7 Fork
  • Ein besseres CMS als Drupal 7
  • Eine Alternative zu Drupal 8

Was spricht für Backdrop?

Backdrop ist durch und durch Responsive. Auch das Backend.

Backdrop vereinfacht die Erstellung neuer responsive Themes. Ein einfach zu bedienender Layout Manager gibt Ihnen unbegrenzte Möglichkeiten, um unterschiedliche Layouts, auf verschiedenen Seiten oder Site sections, einzurichten. Responsive out of the box.

Konfigurationsverwaltung im Core.

Drupal speichert alle Konfigurationen in der Datenbank. Das ändert sich erst mit Drupal 8. Backdrop bietet schon jetzt ein dateibasiertes Management der Konfiguration. Dadurch lassen sich Konfigurationen sehr einfach von einem Staging Server auf den Live Server deployen. Und Konfigurationen können in einer Versionsverwaltung wie git abgelegt werden. So kann man jederzeit zu einer vorherigen Version zurückkehren.

Backdrop ist schneller als Drupal.

Backdrop ist optimiert für MySQL, dem Datenbanksystem, das auf fast jeder Drupal Installation zu finden ist. Backdrop ist schmal und auf das Wesentliche reduziert. Für den Core überflüssige Module, wie das Forum, sind rausgeflogen.

Ein einfacher Umstieg von Drupal 7 ist gewährleistet.

Wenn Sie Drupal 7 kennen, wird Ihnen der Umstieg auf Backdrop leicht fallen. Die selbe PHP Architektur, eine vertraute Inhaltsverwaltung und vieles mehr, das Sie von Drupal 7 kennen, sorgt für einen sorgenfreien Umstieg. Viele Drupal 7 Module lassen sich sehr einfach nach Backdrop portieren.

Weitere Verbesserungen:

  • Wer Drupal kennt, kennt Views. Backdrop hat Views im Core.
  • Backdrop ist komplett in HTML5.
  • Filtersystem der Modulverwaltung
  • AJAX Dialoge
  • Responsive Adminsitration
  • Layouts für Templates.
  • Einfaches UI für Layouts, als Alternative zu Panels.
  • Einfache Dateiverwaltung.
