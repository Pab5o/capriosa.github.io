Sehr interessant waren aber auch die Infos über Aquia Gardens, das geplante Partner Trainingprogramm und über das vielfältige Informations - und Marketingmaterial, welches Acquia Partner zur Verfügung steht. The State of Drupal Wir haben von Dries einiges über die Bedeutung und den Ablauf von Innovationen im Allgemeinen und für Drupal im Besonderen gehört. Die beinahe unüberschaubare Anzahl neuer Features in Drupal 7 wurde kurz vorgestellt. Die verschiedenen Phasen des Code Freeze hat Dries erläutert und einen Ausblick auf Drupal 8 gegeben. Ist Drupal ein Framework oder ein Produkt? Dries meint es sollte ein Framework und Produkt werden. Soviel kurz zum State of Drupal von hier. Geht jetzt mit der nächsten Session weiter. Real time end user theme configuration Wer sich im Vorfeld der DrupalCon dieses Video - http://s3.amazonaws.com/HSCI_Videos/drupalcon_paris.mov - schon mal angeguckt hat, weiß so ungefähr worum es in dieser Session ging. Das gesamte Frontend kann konfiguriert werden. Schriften, Farben, Hintergründe u.v.m.

Also etwas ähnliches wie Acquia mit Drupal Gardens bietet.

Sehr interessante Demonstration was mit der Themes-Settings API, JQuery und der JQuery UI so alles möglich ist. Material zur Session gibts hier: http://maureensmusings.com/node/# Drupal 7 improved theme system Drupal 7 wird es Modul und Theme-Entwicklern wesentlich erleichtern, den finalen Output von Drupal zu verändern. Mit dem hook_page_alter() ist es möglich jeden Teil einer Seite zu verändern bevor er gerendert wird. Zu einem Zeitpunkt, an dem die Seite eigentlich nur ein PHP Array ist, kann alles noch leicht geändert werden. hook_page_alter ist insbesondere für Site builder interessant. Endlich können dann die speziellen Kundenwünsche, was die Änderung des Drupal Outputs betrifft, leicht erfüllt werden. hook_page_alter ermöglicht es jeden Teil einen Page zu beeinflussen. Z.B. einen Teil aus dem Content-Bereich rauszunehmen und in eine Sidebar zu packen.

Zudem gibt es einen neuen Cache. Jeden beliebigen Teil einer Seite kann man jetzt beliebig cachen.

Vielleicht haben wir hier eines der wichtigsten neuen Features von D7 gesehen. OpenAtrium Früh am morgen haben wir unseren Session-Plan geändert. Nicht zur Edipresse, obwohl bestimmt interessant, da wir ja viel für Verlage arbeiten, sondern zur Vorstellung von OpenAtrium. Hierbei handelt es sich um ein Intranet ‘Out of the box’.

Features: Blog Wiki Kalender To do list Chat Casetracker

Da wir in dieser Session aigir kennengelernt haben (ein Drupal hosting system), haben wir auch unseren Session Plan für morgen geändert. Aigir hosting service anstelle von sonnyericson Session. Man kan halt nicht alles haben.

Wichtige Quellen: http://openatrium.com/features http://groups.drupal.org/aegir-hosting-system http://drupal.org/project/features http://features.pingv.com/ http://drupal.org/project/drush_make aegir - Drupal hosting system aegir ist ein Drupal System mit dem sich mehrere Drupal Sites bequem verwalten lassen. Site clones, automatische Backups, updates von Modulen, aber auch auf eine neue Drupal Version sind bequem und vor allem sicher möglich.

aegir erzeugt vor jeder kritischen Aktion ein File und DB-Backup. Zum Anlegen einer neuen Site in aegir erzeugt man einfach einen neuen node, gibt die URL an und wenig später stehen alle Verwaltungs-Tasks von aegir zur Verfügung.

aegir besteht aus folgenden Komponenten: http://drupal.org/project/hosting http://drupal.org/project/provision http://drupal.org/project/hostmaster

Mehr zu aegir hier: http://groups.drupal.org/aegir-hosting-system http://groups.drupal.org/aegir/overview

Bildverwaltung im D7 Core Wir wissen es alle: Bilder mit Drupal hochladen, in Inhalte einzubinden und/oder automatisch verschiedene Bildgrößen zu generieren, kann eine Aufgabe sein, die einen leicht verzweifeln lässt. Nicht, dass dies nicht alles möglich wäre, aber die Möglichkeiten dies zu tun sind kaum noch überschaubar. Zuviele Module für einen Zweck. Image handling in D7 soll uns das Leben erleichtern.

Alles nötige ist in D7 im Core.

File Api FileField ImageAPI ImageCache ImageField Upload - upload.module gibts nicht mehr

Neuer Form Typ - managed_file - steht für die Form API zur Verfügung. Private und public files problemlos möglich. Verzeichnisse pro User möglich Panels 3: was gibt’s Neues Obwohl ich bisher Panels noch nicht eingesetzt habe, wollte ich mir doch mal anschauen was es neues bei Panels 3 gibt. Panels 3 ist eher ein ‘Seiten Management System’ und verändert die Heransgehensweise um Content in Drupal zu positionieren total.

Ich werde es sicherlich mal ausprobieren. Panels 3 auf einer Kundensite einzusetzen, macht nur Sinn, wenn man alle Aspekte genau mit dem Kunden besprochen hat.
